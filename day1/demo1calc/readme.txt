第一个例子,是写了一个加 减的方法,然后写了测试的demo去调用
1.方法被外部包调用方法名必须首字母大写

2.测试的运行不能run方式运行,需要在终端使用go test运行,控制台查看错误信息

3.测试都有一个参数 *testing.T 这个参数是为了抛出异常的

4.测试类文件名需要加上下划线(例:aa_test);test测试方法,首字母要大写,否则调用不了

5.t.Log()只有在使用详细日志时候会打印出来(go test -v),否则只能知道测试方法是否有异常

6.使用终端,目录应该是calc_test.go测试类的上一级(test目录),使用go test运行测试,如果想查看详细信息,就使用go test -v
