第一个例子,是写了一个加 减的方法,然后写了测试的demo去调用
1.调用不是本包的方法 有一个导包的操作

2.调用其他包中方法需要用包名去调用(不是去文件名)
   test下的测试 调用 demo1calc 包下的calc.go文件里面的sum方法
   我使用calc.Sum发现不能调用,这是错误的调用方式,应该是包名demo1calc.Sum
   方法被外部包调用方法名必须首字母大写

3.测试的运行不能run方式运行,需要在终端使用go test运行,控制台查看错误信息
如果觉得测试结果不完整,可以使用go test -v查看详细错误信息

4.测试都有一个参数 *testing.T 这个参数是为了抛出异常的
5.测试中错误信息使用Fatal方法抛出,日志使用Logf方法抛出,参数都是string字符串

6.test测试方法,首字母要大写,否则调用不了

7.t.Log()只有在使用详细日志时候会打印出来(go test -v),否则只能知道测试方法是否有异常
